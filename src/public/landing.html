SmartLocker Backend
API Reference

A lightweight REST API powering the SmartLocker network — handling stations, packages, delivery agents, and customer pickups.

Base URL Preferred: /api
Landing and health are shown in their direct form without /api.
Backend
Latest Route Map
Most endpoints are listed with the /api prefix. Landing and health are shown without the prefix for quick checks.
Public & Auth
GET / Serves this API landing page.
GET /health Returns service status and database connectivity.
POST /api/auth/signup Create customer account.
POST /api/auth/signup/admin Create admin account.
POST /api/auth/login Customer login and JWT issuance.
POST /api/auth/login/admin Admin login and JWT issuance.
GET /api/auth/session Return current authenticated session context.
Admin & Agent
GET /api/admin/users List users (admin only).
PUT /api/admin/users/:userId/reset-password Reset user password (admin only).
GET /api/stations List stations (admin).
GET /api/agent/stations List stations visible to delivery agent.
GET /api/lockers List lockers (admin, delivery agent).
POST /api/lockers Create locker (admin).
GET /api/packages List packages for delivery agent.
PUT /api/packages/assign-locker Assign a locker to package (delivery agent).
PUT /api/packages/store Store package in assigned locker (delivery agent).
Customer
GET /api/customer/packages List customer packages and states.
GET /api/customer/bills List customer storage bills and pricing details.
POST /api/locker/unlock Validate unlock and preview retrieval charge.
POST /api/locker/retrieve/confirm Confirm retrieval and finalize package pickup.
GET /api/notifications List customer notifications.
PUT /api/notifications/:notificationId/read Mark a notification as read.
Responses
HTTP Status Codes
200OK — request succeeded
201Created — resource created
400Bad Request — missing fields
401Unauthorized — invalid code
404Not Found — no such resource
409Conflict — already processed
